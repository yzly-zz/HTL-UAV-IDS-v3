# 在线演示页提交建议

“作品在线演示页面链接”在你给出的要求里是可选项，不是必填项。所以最保守的做法是：

- 必交三项先做扎实：设计文档、源代码 ZIP、演示视频
- 在线链接来不及就可以不交

如果你想补一个稳定链接，不能依赖你自己的电脑一直开机，应该把前端部署到云端静态托管。

## 当前项目适合的做法

项目前端在 `src/web_gcs`，我已经把它改成：

- 优先连接实时后端
- 连不上时自动切到内置演示模式
- 也可以通过 URL 参数 `?demo=1` 强制进入演示模式

这意味着你现在可以只部署前端静态页，不部署本地 Python 后端，也能给评委一个能打开、能交互、能看到动态效果的在线演示页。

## 推荐托管平台

- GitHub Pages
- Vercel
- Netlify
- Cloudflare Pages

这几种都不需要你本机持续运行。

## 本地构建

进入目录：

```powershell
cd F:\Project\HTL-UAV-IDS-V3\src\web_gcs
```

执行：

```powershell
npm run build
```

产物目录：

```text
F:\Project\HTL-UAV-IDS-V3\src\web_gcs\dist
```

把 `dist` 上传到任意静态托管平台即可。

## 如果后面还想接真实后端

部署平台可配置环境变量：

```text
VITE_WS_URL=wss://你的后端域名/ws/traffic
```

不配置也没关系，页面会自动降级到演示模式。

## 比赛提交建议

- 如果交在线链接，建议直接交静态演示页链接
- 页面标题、说明文字、仓库名、页脚都不要出现学校、团队、姓名、教师等身份信息
- 如果你用 GitHub Pages，仓库公开页和 README 也要做匿名检查
